Designing a Real-Time Auction Platform

30-Second Interview Explanation

For a real-time auction platform, I would use a clear event-driven flow. REST APIs handle setup, like creating auctions, logging in, and retrieving history. Socket.IO handles live bids. The server is authoritative, so clients only submit intent. The backend validates bid amounts, auction timing, and eligibility and never trusts the client. Clients send bids to the Node server they are connected to. In a horizontally scaled setup with multiple servers, each server checks the current auction state in Redis, which provides fast in-memory reads and keeps all servers consistent without overloading the database. When a bid is being saved, Postgres row-level locks ensure only one bid updates at a time. If the bid is valid, the server updates Postgres, updates Redis with the new highest bid, and broadcasts that update to all clients. Other Node servers read from Redis so all connected clients see the same bid. Transactions re-validate to prevent race conditions and keep state consistent. Broadcasting is per-auction. Sockets join auction-specific rooms, and io.to(room).emit() sends updates only to clients in that room. Testing covers unit checks for processBid logic and integration tests for REST endpoints and Socket.IO flows, including authentication.

High-Level Architecture

  • Frontend: Next.js (React)
  • Backend: Node.js (Express or NestJS)
  • Realtime: Socket.IO
  • Database: PostgreSQL (durable storage, row-level locks for single-instance concurrency)
  • Cache: Redis (fast in-memory cache for bid reads)

Communication Split

Purpose Technology
Create auctions, login, history REST API
Live bid updates WebSockets (Socket.IO)

Core Models (Normalized)

  • User – stores bidder or admin information (id, name, credentials, role)
  • Auction – represents a single auction (id, startTime, endTime, status, itemId, highestBidId)
  • Item – the thing being auctioned (id, name, description, metadata)
  • Bid – individual bid (id, auctionId, userId, amount, timestamp)
  • AuctionStatus – enum or reference table (OPEN, CLOSED, PENDING, CANCELLED)

Real-Time Flow

1. Client Joins Auction, Submits Bid, and Receives Real-Time Updates

    
    import { useEffect, useState } from "react";
    import { io } from "socket.io-client";
    
    export default function AuctionPage({ auctionId }) {
      const [socket, setSocket] = useState(null);
      const [highestBid, setHighestBid] = useState(null);
      const [bidAmount, setBidAmount] = useState("");
    
      useEffect(() => {
        // Create socket connection
        const newSocket = io("https://api.example.com", {
          auth: {
            token: localStorage.getItem("jwt")
          }
        });
    
        setSocket(newSocket);
    
        // CLIENT-SERVER: emit joinAuction → server adds socket to room
        newSocket.emit("joinAuction", auctionId);
    
        // CLIENT-SERVER: highestBidUpdated event maps to setHighestBid (server is authoritative)
        newSocket.on("highestBidUpdated", (data) => {
          setHighestBid(data.highestBid);
        });
    
        // Cleanup on unmount
        return () => newSocket.disconnect();
      }, [auctionId]);
    
      const handlePlaceBid = () => {
        if (!socket) return;
    
        // CLIENT-SERVER: placeBid is intent only; server validates and broadcasts result
        socket.emit("placeBid", {
          auctionId,
          amount: Number(bidAmount)
        });
      };
    
      return (
        <div>
          <h2>Highest Bid: ${highestBid}</h2>
    
          <input
            type="number"
            value={bidAmount}
            onChange={(e) => setBidAmount(e.target.value)}
          />
    
          <button onClick={handlePlaceBid}>
            Place Bid
          </button>
        </div>
      );
    }

2. Server Validates Bid and Updates Auction State


const io = new Server(httpServer, {
  cors: {
    origin: "https://your-frontend.com",
    credentials: true
  }
});

// AUTH: io.use() runs before connection; only verified users reach io.on("connection")
io.use((socket, next) => {
  try {
    const token = socket.handshake.auth.token;
    if (!token) return next(new Error("Unauthorized"));

    const user = verifyJWT(token);
    socket.user = user;
    next();
  } catch (err) {
    next(new Error("Unauthorized"));
  }
});

io.on("connection", (socket) => {

    // BROADCAST: socket.join(room) → io.to(room).emit() sends only to clients in that auction
    socket.on("joinAuction", (auctionId) => {
        socket.join(`auction:${auctionId}`);
    });
    
    socket.on("placeBid", async ({ auctionId, amount }) => {
    
        // 1. Authenticate user (JWT validated earlier in connection middleware)
        const user = socket.user;
    
        // 2. REDIS: Load current auction state from cache for high-throughput bid reads
        //    Redis = fast in-memory; avoids hitting Postgres on every bid. Optional: Redis WATCH/MULTI
        //    for atomic ops when running multiple Node instances (multi-process concurrency).
        //    const auction = await redis.get(`auction:${auctionId}`);
    
        // 3. Validate auction status and bid amount
        //    - Auction is still ACTIVE
        //    - Bid is higher than current highest
        //    - User is eligible to bid
    
        // CONCURRENCY: processBid uses Postgres row-level lock (SELECT FOR UPDATE) for single-instance.
        // Multi-instance: use Redis WATCH/MULTI or queue for sequential bid processing.
    
        // 5. POSTGRES: Durable persistence; Postgres = source of truth. Redis cache updated after.
        //    - Insert into BidHistory
        //    - Update Auction.highestBid
    
        const updatedAuction = await processBid(auctionId, user.id, amount);
    
        if (!updatedAuction.success) {
        socket.emit("bidRejected", { reason: updatedAuction.reason });
        return;
        }
    
        // BROADCAST: io.to(room) sends only to clients in this auction. Multi-server: Socket.IO
        // Redis Pub/Sub adapter relays emit across Node instances so all rooms get updates.
        io.to(`auction:${auctionId}`).emit("highestBidUpdated", {
        highestBid: updatedAuction.highestBid,
        highestBidder: user.id
        });
    
    });
});

3. Validation and Concurrency Handling


async function processBid(auctionId: string, userId: string, amount: number) {
    // 1. POSTGRES: Source of truth. For high throughput, read from Redis cache first;
    //    fall back to DB or use Redis for hot path, Postgres for durability.
    const auction = await db.auctions.findOne({ id: auctionId });
    
    // 2. Validate auction status and bid amount
    if (!auction || auction.status !== "open") {
        return { success: false, reason: "Auction not open" };
    }
    if (amount <= auction.highestBid) {
        return { success: false, reason: "Bid too low" };
    }
    
    // CONCURRENCY: POSTGRES row-level lock (single-instance). Multi-instance: use Redis
    // WATCH/MULTI or a queue for sequential processing across processes.
    const updatedAuction = await db.transaction(async (tx) => {
        // CRITICAL: SELECT FOR UPDATE locks row until transaction commits
        const current = await tx.auctions.findOne({ id: auctionId }, { lock: "FOR UPDATE" });
    
        // Re-validate inside transaction to catch race conditions
        if (amount <= current.highestBid) {
            throw new Error("Bid too low");
        }
    
        // POSTGRES: Durable write; Redis cache would be invalidated/updated after commit
        const result = await tx.auctions.update({ id: auctionId }, {
            highestBid: amount,
            highestBidder: userId
        });
    
        // Optionally write bid history / audit logs here
        await tx.bidHistory.insert({
            auctionId,
            bidderId: userId,
            amount,
            timestamp: new Date()
        });
    
        return result;
    });
    
    // BROADCAST: io.to(room) targets only clients in this auction. With multiple Node
    // instances, Socket.IO Redis Pub/Sub adapter ensures emit reaches all instances' rooms.
    io.to(`auction:${auctionId}`).emit("highestBidUpdated", {
        highestBid: updatedAuction.highestBid,
        highestBidder: userId
    });
    
    // 5. Return success and updated state
    return { success: true, highestBid: updatedAuction.highestBid };
    }

Auction End Handling

  • Mark auction as CLOSED
  • Persist final bid
  • Emit auctionEnded event
  • Disable client bidding

Security Considerations

  • Never trust client bid amount
  • Server controls auction timing
  • Authenticate all socket connections
