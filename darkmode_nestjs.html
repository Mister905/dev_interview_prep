NestJS Fundamentals Cheat Sheet

1. Modules

  • Organize the application into logical units.
  • Declare providers and controllers inside a module.
  • Export providers to share them with other modules.
@Module({
  imports: [],
  controllers: [MyController],
  providers: [MyService],
  exports: [MyService]
})
export class MyModule {}

2. Providers

  • Classes that encapsulate business logic or utilities.
  • Marked with @Injectable() and registered in a module.
  • Can be injected into controllers or other providers.
@Injectable()
export class MyService {
  doSomething() {
    // business logic
  }
}

3. Controllers

  • Handle incoming requests and return responses.
  • Delegate all heavy logic to providers.
  • Define HTTP routes using decorators like @Get(), @Post(), etc. 
@Controller('example')
export class MyController {
  constructor(private readonly myService: MyService) {}

  @Get()
  getExample() {
    return this.myService.doSomething();
  }
}

4. Dependency Injection (DI)

  • NestJS injects providers automatically when registered in a module.
  • Enables nested services and clean separation of concerns.
constructor(private readonly myService: MyService) {}

5. Decorators

  • @Module() → defines a module.
  • @Injectable() → marks a class as a provider.
  • @Controller() → defines a controller.
  • @Get(), @Post(), @Put(), @Delete() → define HTTP routes.
  • @Inject() → explicitly inject a provider.

6. Optional Concepts

  • Pipes: transform or validate input data (e.g., ValidationPipe).
  • Guards: handle authorization (e.g., AuthGuard).
  • Interceptors: modify responses, logging, or error handling.

7. Bootstrapping the App

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);
}
bootstrap();
