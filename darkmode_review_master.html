Frontend Fundamentals

HTML & CSS

What is semantic HTML?

Uses elements like <article>, <section>, and <nav> to give meaning to content, improving accessibility, SEO, and code clarity.

How do you use <label> for accessibility?

Link <label> elements to form controls to improve screen reader support and make click targets easier for users.

What are component libraries and design systems?

Reusable UI components and guidelines that ensure consistency across applications and devices while speeding up development.

How does CSS specificity and conflict resolution work?

Inline styles override IDs, which override classes, which override elements. Avoid !important and use DevTools to debug conflicts.

What is the CSS box model?

Content, padding, border, and margin. Padding adds space inside; margin adds space outside.

How do you create responsive layouts?
  • Media queries, fluid units, Flexbox, and CSS Grid
  • Frameworks like Tailwind or Bootstrap for faster development

What are the differences between em, rem, and px?

px is fixed; em is relative to parent font size; rem is relative to root font size, maintaining consistent scaling.

JavaScript

What are the differences between var, let, and const?
  • var is function-scoped; let and const are block-scoped
  • const for values that don't change; let for reassignable variables

How do you handle async code?

Use async/await with try/catch for readable asynchronous code and error handling.

What are primitives vs non-primitives?

Primitives like strings and numbers are compared by value; objects and arrays are compared by reference, affecting React re-renders.

How do you ensure data integrity on frontend and backend?
  • Frontend validation improves user experience
  • Backend validation ensures security and prevents malicious data

How do you sanitize input?

Remove or escape harmful input on both frontend and backend to prevent SQL injection or XSS attacks.

What is the difference between == and ===?

== checks equality with type coercion; === checks both type and value. Always use === to avoid unexpected conversions.

What are closures?

Functions that retain access to variables from their outer scope after the outer function finishes, enabling private state and encapsulation.

What are promises?

Represent asynchronous operation results. Handle success with .then() and errors with .catch(), or use async/await for cleaner syntax.

What is hoisting?

JavaScript moves variable and function declarations to the top of their scope. var can be accessed before definition (undefined); let and const cannot.

What are arrow functions?

Shorter function syntax that automatically inherits this from the surrounding scope, simplifying context handling.

How does the 'this' keyword work?

Refers to the function's call context. As an object method, this points to that object; standalone, it points to global or is undefined in strict mode.

React

How do you design React components?
  • Small, reusable, and accessible
  • Manage side effects carefully
  • Use memoization to prevent unnecessary re-renders

How do you manage complex state?
  • useState for local component state
  • Redux for global app-wide state
  • Formik for form state and validation

How do you prevent performance issues?
  • Memoization to avoid unnecessary re-renders
  • Lazy loading for components and data
  • Pagination or virtualization for large datasets

What is the difference between controlled and uncontrolled components?

Controlled components sync value with React state for full control. Uncontrolled components manage internal state and are accessed via refs.

How do you use useEffect?
  • Handle side effects like data fetching or subscriptions
  • Manage dependency array carefully
  • Include cleanup functions to avoid memory leaks

How do you make API calls in React?
  • useEffect with axios for component data fetching
  • Redux async thunks for centralized API calls in larger apps

Why are keys important in lists?

Let React identify which items changed, enabling efficient updates and preventing unnecessary re-renders.

What is the Virtual DOM and reconciliation?

React's lightweight DOM representation. Reconciliation compares versions and updates only changed parts of the real DOM, improving performance.

What is the difference between state and props?

State is mutable and managed within a component; props are read-only values passed from parent to child.

How do you do conditional rendering?

Use ternary operators, logical &&, or helper components to render content based on state or props.

How do you use React Router?

Handle navigation and dynamic routes in single-page applications, allowing page changes without full reloads.

What is the difference between Next.js SSR and SSG?

SSR generates pages on each request for fresh data. SSG pre-renders at build time for faster delivery when data doesn't change often.

What performance issues have you encountered on the frontend, and how did you fix them?
  • Code-split routes and lazy load components for slow initial loads
  • Use React.memo, useMemo, and useCallback to prevent re-renders
  • Debounce and cache API calls; use performance monitoring tools

Explain state management in a frontend app. When is global state necessary?

Use local state for component-specific data; global state when multiple components or pages need the same data, like authentication or theme settings.

How do you handle forms, validation, and user feedback?
  • Controlled components with client and server validation
  • Formik or React Hook Form for complex forms
  • Inline errors, loading states, and success messages

What causes unnecessary re-renders, and how do you prevent them?
  • Creating new objects/arrays in render or passing inline functions
  • Prevent with useMemo, useCallback, React.memo, and moving state down
  • Use React DevTools Profiler to identify sources

How do you make an app accessible?
  • Semantic HTML, ARIA labels, keyboard navigation
  • Proper focus management and screen reader testing
  • Follow WCAG guidelines

UX / UI Engineering

How do UX decisions impact backend or database design?
  • Real-time features need WebSockets; search requires indexing
  • Pagination affects API design; permissions shape authorization logic
  • Loading states may need partial data endpoints

How do you ensure consistency in UI components?
  • Design system or component library with documented patterns
  • Reusable components with consistent props and styling
  • Code reviews and Storybook to maintain consistency

How do you balance usability vs performance?
  • Focus performance on core user flows
  • Use progressive enhancement, loading states, skeleton screens
  • Measure with Lighthouse and gather user feedback

How do you design forms to minimize user errors?
  • Clear labels, helpful placeholders, appropriate input types
  • Real-time validation with immediate error feedback
  • Disable submit until required fields are valid

How do you handle loading states, errors, and empty states?
  • Skeleton screens show layout while content loads
  • Errors display clear messages with next steps and retry options
  • Empty states explain missing content and suggest actions

TypeScript

Why use TypeScript?
  • Static typing catches bugs early
  • Improves maintainability and IDE support with autocomplete
  • Helps teams work more efficiently

What is the difference between type and interface?

Interfaces define object shapes and can be extended. Types are more flexible, defining unions, tuples, and primitives. Interfaces preferred for extensible objects.

What are generics?

Create reusable functions or components that work with different types while keeping type safety, avoiding code duplication.

What is type inference?

TypeScript automatically figures out variable or expression types. Add explicit types at public APIs or where clarity matters.

Backend & Infrastructure

How do you design a Node.js API?
  • RESTful endpoints around clear resources
  • Thin controllers, business logic in services
  • Data access layer or ORM; consistent responses and error handling

How do you design a REST API? What makes one "good"?
  • Clear resource-based URLs with proper HTTP verbs
  • Stateless, cacheable, versioned, and documented
  • Predictable error handling, pagination, filtering; focus on client needs

How do you handle errors and ensure reliability in Node?
  • Validate inputs early, return structured error responses
  • Centralize error handling with enough context for troubleshooting
  • Design services to fail gracefully instead of crashing

How do you think about scalability and performance?
  • Optimize database queries and use caching
  • Organize system for smooth growth
  • Monitor and fix slow parts before they affect users

How do you structure a Node.js project?
  • Organize by feature: routes, controllers, services, data access separate
  • Shared utilities and configs in their own folders
  • Clear naming for maintainability and scalability

What's the difference between blocking and non-blocking code in Node?

Blocking code stops other requests until it finishes. Non-blocking code handles multiple requests simultaneously using async patterns, keeping the server responsive.

When would you choose REST vs GraphQL?

REST for simple CRUD and predictable caching. GraphQL when clients need flexible queries and to avoid over-fetching.

How do you validate and sanitize user input?
  • Check structure, types, and required fields using Joi or Zod
  • Escape harmful characters and use parameterized queries
  • Validate on both client and server; prefer whitelisting

Explain middleware and when you'd use it.
  • Functions that run between request and response
  • Use for authentication, logging, error handling, request parsing, CORS
  • Keep route handlers focused on business logic

How do you structure a backend project for maintainability?
  • Separate routes, controllers, services, and repositories
  • Small, focused functions with consistent naming
  • Testing, documentation, and code reviews ensure maintainability

What are common backend performance bottlenecks?
  • Slow queries, N+1 problems, inefficient algorithms, blocking I/O
  • Identify with profiling and monitoring
  • Optimize queries, add indexes, implement caching, scale horizontally

How do you handle authentication?
  • Token-based authentication with JWTs, validated via middleware
  • Refresh tokens for session continuity
  • Role-based access control and secure secret storage

How do you test APIs?
  • Postman for manual and exploratory testing
  • Jest for automated integration and unit tests
  • Focus on behavior: verify responses, status codes, and side effects

How do you deploy Node services?
  • CI/CD pipeline builds, tests, and deploys to cloud
  • Environment variables for configuration; stateless services
  • Monitor deployments and implement rollback procedures

How do you design a database schema?
  • Understand key entities and relationships
  • Normalize tables, choose appropriate data types
  • Enforce integrity with constraints and indexes; adjust based on usage

How do you optimize database performance?
  • Optimize only when needed: slow queries or over-fetching
  • Use indexes and write efficient queries with pagination
  • Monitor and analyze query performance to guide optimization

How do you use ORMs effectively?
  • Use for consistent, maintainable data access
  • Drop to raw queries for performance-critical operations

When would you choose SQL vs NoSQL?

SQL when relationships, transactions, and consistency matter. NoSQL for flexibility, horizontal scaling, or document-based storage.

Explain indexing and how it improves performance.
  • Indexes let databases quickly find rows without scanning the whole table
  • Speed up searches, joins, and sorting; can slow down writes
  • Add to frequently queried columns and foreign keys; avoid over-indexing

What is normalization? When might you not normalize?
  • Organizes data into related tables to reduce redundancy
  • Denormalize when joins are expensive or read speed is critical
  • Trade storage space for faster reads

How do you handle migrations and schema changes?
  • Versioned migration files applied in order
  • Test in staging; for risky changes, add new column, migrate data, update code, then remove old
  • Backup before migrations; have rollback plan

Describe a slow query you've optimized.

Added indexes on join and filter columns and restructured a subquery, reducing execution time from 5 seconds to under 100 milliseconds.

How do transactions work, and when are they important?
  • Group multiple operations so all succeed or all fail
  • Essential for operations that must happen together, like fund transfers
  • Prevent inconsistent or corrupted data

How do you deploy and run services in the cloud?
  • Repeatable build and deployment steps
  • Separate environments; stateless services for scaling
  • Use load balancers and scaling strategies as needed

How do you approach CI/CD pipelines?
  • Automatically run tests, build, and deploy
  • Ensure consistency across environments
  • Catch issues early; keep deployments reliable and repeatable

How do you handle configuration and secrets?

Keep out of codebase; manage with environment variables for safe, secure, environment-independent deployments.

How do you monitor and respond to issues in production?
  • Use logging and monitoring tools to track behavior
  • Reproduce, identify root cause, apply fix
  • Document learnings to prevent future occurrences

How do you think about scaling?
  • Stateless services for easy instance addition
  • Efficient data access, load balancing, caching
  • Async processing; add complexity only when needed

DevOps & Tools

What is CI/CD, and how do you approach it?
  • Automate building, testing, and deploying code
  • Run linting, builds, and tests on every commit
  • Promote approved changes consistently
  • Faster feedback, fewer manual errors, reliable deployments

What is your experience with cloud and Docker?
  • Separate frontend and backend in Docker containers
  • Orchestrate with Docker Compose
  • Isolates dependencies, ensures consistent environments

How do you use Git?

Version control for tracking code changes. Use daily for branching, pull requests, code reviews, and resolving conflicts.

How do you approach testing?
  • Jest for unit and integration tests
  • Postman to validate API endpoints

What is your experience with Linux?

Comfortable with command line for navigation, file management, permissions, and basic editing.

Walk me through how you'd deploy a full-stack app.
  • Deploy frontend and backend separately through CI pipelines
  • Configure with environment variables
  • Use staging environments, health checks, monitoring, and rollback strategies

What are environment variables, and why do they matter?
  • Configuration values at runtime, like API keys or database URLs
  • Keep secrets out of source control
  • Enable same code to run in different environments

Testing & Quality

What types of tests do you write (unit, integration, e2e)?
  • Many unit tests for isolated logic
  • Some integration tests for components and APIs
  • Few E2E tests for critical user flows

How do you decide what's worth testing?
  • Prioritize business logic, critical user flows, and edge cases
  • Avoid trivial code, implementation details, and third-party libraries
  • Focus on high-value tests that catch real bugs

System Design

How would you design a simple chat application?
  • WebSockets for real-time messaging
  • Backend routes messages, tracks online/offline users, stores in indexed database
  • Message queue for scaling across multiple servers
  • Frontend handles reconnections and displays messages in real-time

How would you handle file uploads at scale?
  • Cloud storage like S3 or Azure Blob Storage
  • Backend generates secure upload tokens for direct client-to-cloud uploads
  • Chunk large files; validate types and sizes
  • Serve through CDN; process with background jobs
  • Database stores only metadata, not files

How would you design an API for a to-do app used by thousands of users?
  • RESTful endpoints with pagination and filtering
  • JWT authentication; indexes on user_id and created_at
  • Rate limiting and caching for scaling

How would you prevent duplicate requests or race conditions?
  • Frontend: disable buttons during submission, send unique request keys
  • Backend: enforce database constraints, use transactions and locking
  • Ensure each operation happens exactly once
