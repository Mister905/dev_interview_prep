System Design Glossary

Client-Server Architecture

  • Client: Frontend app (web, mobile) sending requests.
  • Server: Backend machine handling requests and returning responses.
  • IP Address: Unique server identifier.
  • DNS: Maps domain names to IPs for human-friendly access.
  • Latency: Delay due to distance between client and server; reduced via geographically distributed data centers.

HTTP & HTTPS

  • HTTP: Protocol for client-server communication; includes request headers, body, and response.
  • HTTPS: Encrypted HTTP using SSL/TLS.

APIs

  • API: Middleware for structured communication between client and server.
  • REST: Stateless, resource-based, uses standard HTTP methods.
  • GraphQL: Allows clients to request exactly the data they need; reduces over-fetching.

Databases

  • SQL: Structured schema, ACID compliance; ideal for relational data.
  • NoSQL: Flexible schema, high scalability; includes key-value, document, graph, and wide-column stores.
  • Indexing: Improves read speed using lookup tables; trade-off: slower writes.
  • Replication: Primary for writes, multiple read replicas for scaling and high availability.
  • Sharding (Horizontal Partitioning): Split database by rows using a sharding key to distribute load.
  • Vertical Partitioning: Split database by columns to optimize queries.
  • Normalization: Reduces redundancy; may require joins.
  • Denormalization: Combines related data to reduce joins; improves read-heavy performance.

Scaling

  • Vertical Scaling: Upgrade single server (CPU, RAM); limited capacity, expensive.
  • Horizontal Scaling: Add more servers to share load; improves reliability and capacity.
  • Load Balancer: Distributes traffic across servers; uses algorithms like round-robin, least connections, IP hashing.

Caching

  • Cache: Store frequently accessed data in memory for faster reads.
  • Cache-Aside Pattern: Application checks cache first, then database; stores result in cache.
  • TTL (Time-To-Live): Expiration to avoid stale data.

Content Delivery

  • Blob Storage: Store unstructured files (images, video) in cloud buckets; each file gets a unique URL.
  • CDN: Distributed servers delivering content from closest location; reduces latency and buffering.

Real-Time Communication

  • WebSockets: Persistent, two-way client-server communication for real-time updates.
  • Webhooks: Server-to-server notifications triggered by events; avoids polling.

Architecture Patterns

  • Monolithic: Single large codebase; simple for small apps but hard to scale.
  • Microservices: Independent services handling specific responsibilities; communicate via APIs or message queues; scalable and deployable individually.

Message Queues

  • Producer: Sends message to queue.
  • Queue: Temporarily holds messages.
  • Consumer: Processes messages asynchronously.
  • Decouples services and improves scalability.

Rate Limiting & API Management

  • Rate Limiting: Restrict requests per user/IP to prevent overload.
  • Algorithms: Fixed window, sliding window, token bucket.
  • API Gateway: Central entry point; handles auth, rate limiting, routing, logging.

Idempotency

  • Ensures repeated requests produce same result.
  • Use unique request IDs to detect duplicates.

Distributed Systems

  • CAP Theorem: A distributed system can have at most two of Consistency, Availability, Partition Tolerance.
  • Design choice depends on system requirements.
