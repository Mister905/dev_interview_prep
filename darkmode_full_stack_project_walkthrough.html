Project-Agnostic Step-by-Step Workflow

Step 0 — Project Initialization

  • Plan the architecture: Define routes, modules, API endpoints, database schema, and domain entities (e.g., User, Item, Record).
  • Create repository structure: 
    /frontend        # Next.js App Router, pages, components, Redux
/backend         # NestJS modules, Prisma schema, controllers/services
/infra           # Cloud config, deployment scripts
.gitignore
docker-compose.yml
README.md
  • Set up version control: Git repo with branching strategy (main/dev/feature)
Manual test: git status works and initial commit is pushed.

Step 1 — Backend Setup (NestJS + Prisma + Postgres/Supabase)

  • Initialize NestJS: nest new backend
  • Install dependencies: @nestjs/config, @nestjs/jwt, @prisma/client, class-validator, supabase-js
  • Backend structure: 
    /backend
├─ src/
│   ├─ main.ts           # bootstrap app
│   ├─ app.module.ts     # global modules
│   ├─ auth/
│   │   ├─ auth.module.ts
│   │   ├─ auth.guard.ts
│   │   └─ supabase.service.ts
│   ├─ user/
│   │   ├─ user.module.ts
│   │   ├─ user.controller.ts
│   │   └─ user.service.ts
│   └─ core/
│       ├─ core.module.ts
│       ├─ entity.controller.ts
│       └─ entity.service.ts
├─ prisma/
│   ├─ schema.prisma
│   └─ migrations/
├─ Dockerfile.prod
└─ package.json
  • Configure Prisma: prisma init, define models
  • Create database and run migrations
  • Implement AuthGuard for JWT verification
  • Create DTOs and validation pipes
  • Implement core services and controllers (CRUD endpoints)
Manual test:
  • Run backend → GET /api/health returns 200
  • Create record via API → DB row exists
  • JWT verification works for protected endpoints

Step 2 — Frontend Setup (Next.js App Router)

  • Initialize Next.js: npx create-next-app frontend
  • Install dependencies: Redux, Supabase client, Axios or fetch wrapper
  • Frontend structure: 
    /frontend
├─ app/
│   ├─ layout.tsx
│   ├─ page.tsx
│   ├─ [id]/
│   │   ├─ page.tsx
│   │   └─ EntityDetailClient.tsx
│   └─ auth/
│       ├─ login/page.tsx
│       └─ callback/page.tsx
├─ components/
├─ lib/
│   ├─ api/client.ts
│   └─ auth/supabase.ts
├─ store/
│   └─ entitySlice.ts
├─ styles/
│   └─ globals.css
├─ public/
│   └─ images/
└─ next.config.js
  • Configure Redux store and slices
  • Add auth client wrapper for login/session handling
  • Implement login page connecting to backend via JWT
Manual test:
  • Run yarn dev → login page loads
  • Submit login → token stored, user redirected
  • API calls with Authorization header work

Step 3 — Implement Core Domain Logic

  • Backend:
    • createEntity() → lightweight record creation
    • startProcess() → heavier setup/related records, update status
    • updateEntity() → handle mutations/updates
    • Optional finalization logic → mark record as completed or archived
  • Frontend: List and detail views, client-only selection, API calls for create/update/finalize
Manual test: Create → start → update → finalize workflows all reflect in DB and UI.

Step 4 — Infrastructure & Deployment

  • S3 + CloudFront: configure static frontend export and SPA routing
  • ECS Fargate: deploy backend container, secrets from SSM or environment
  • CI/CD: GitHub Actions for frontend and backend
Manual test:
  • Frontend loads via CloudFront
  • Backend endpoints reachable via /api
  • SPA routes handled correctly

Step 5 — Testing & Validation

  • Manual tests: auth flow, entity lifecycle, state transitions, unauthorized access, CORS
  • Optional automated tests: Jest/Supertest (backend), Cypress/Playwright (frontend)

Step 6 — Optimizations / Extras

  • Logging and monitoring
  • Error handling and validation
  • Check CI/CD environment variables
  • Optimize Prisma queries
  • Document architectural decisions (Next.js vs Vite, static export vs SSR, state management)

Suggested Project Order Summary

  1. Plan architecture & DB schema
  2. Initialize backend → modules → Prisma → Auth → DTOs
  3. Initialize frontend → Redux → Auth client → pages/components
  4. Implement core domain logic (entity lifecycle) on backend + frontend
  5. Deploy backend (ECS) and frontend (S3 + CloudFront)
  6. Manual + automated testing of auth, entity lifecycle, state machine, API
  7. CI/CD setup and environment validation
  8. Optimization, logging, monitoring, and documentation
